About Project

The bugzilla data is in Metric's ElasticSearch container, and is updated a few times daily. My goal is to provide metrics and numbers to development teams tha can not be easily pulled form Bugzilla

Bugzilla Anthropology Metrics Prototypes

Charts

Burndown.html Open Bug Count by Program, Product, Component

Burnup.html Closed Bug Count by Program, Product, Component

NewAndClosed.html Show weekly totals for new bugs and closed bugs (new bugs by Program are artificailly low because not marked upon creation)

BugAge.html FAST! - Age of closed bugs - Uses ES term packing

TimeToResolution.html SLOW (about 30sec) - Same as bug age, only measurment stars when bug is marked as belonging to Program, Product or Component

ReviewTimes.html SLOW (about 3min) - Length of time a bug waits for a review

ResolutionsOverTime.html FAST! - Show number of each Resolution by the age of the bug. This query packs mutiple data partitions into a single "term" facet

Data

Reviews_Pending.html List of ALL bugs pending review, grouped by requestee

Reviews_Pending_Core.html List of reviews for Firefox and Core, and only requested in the last year

Reviews_NoReviewer.html List of all bugs with no reviewer

Reviews_Basecamp_Only.html Andrew's Basecamp Blockers - but using the ES documnet store instead
